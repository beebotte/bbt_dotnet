Channel
A Channel represents a logical entity that can be (1) a physical object as a demotic sensor connected to an Arduino or a RaspBerry Pi, or, (2) a virtual object as a Blog, Twitter account, web service, etc. A channel belongs to one and a unique account, it is expected to connect to Beebotte to write data. It can also connect to read data from the platform. A channel can be created by the user; it has a unique name that will be used to identify it, a description, an array of resources and an access level indicating if it is public or private. The channel concept in Beebotte is rather abstract. It is required to be connected to the Internet (through Ethernet, WiFi, 3G, ZigBee, etc.) and expected to read and write to the Beebotte Cloud platform.
Connector
The Connector is the interface that makes it possible for applications to communicate with Beebotte. The Connector allows you to Read, Create and Delete Channels and Resources, Read data from Beebotte, and Write and Publish data to Beebotte.

Resource
A resource belongs to a channel, it provides the data to record in the platform. For example, a temperature sensor of a weather station (channel) will emit a temperature event (resource) to record in the platform. A resource can be an attribute like the temperature, the humidity level, etc. It can also be an event like to indicate low battery level. It can also be a function like a face identification service where the user sends an image and expects the number of human faces in return.
